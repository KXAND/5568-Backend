# Git Rules

Read this file before you start coding in this repository.

Ask AI and learn Git Here: [learngitbranching](https://learngitbranching.js.org/?locale=zh_CN)

## Team Rules (Must Follow)

1. Pull the latest code before starting work: `git pull`.
2. Create your own branch for each task. Do not work directly on `main`.
3. Keep each commit focused on one change.
4. Write clear commit messages.
5. Run compile before pushing.
6. Never force-push (`git push --force`) to shared branches.
7. Resolve conflicts locally and verify the project still runs before pushing.
8. If a change affects shared files, notify the team in chat before large edits.
